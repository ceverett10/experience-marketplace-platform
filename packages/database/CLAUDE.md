# Database Package (`@experience-marketplace/database`)

Prisma ORM for PostgreSQL. Central data layer for the entire platform.

## Commands

```bash
npm run db:generate --workspace=@experience-marketplace/database   # Generate client
npm run db:migrate --workspace=@experience-marketplace/database    # Run migrations
```

## Connection

Singleton `PrismaClient` with pool limit 4 (Heroku Postgres essential-1).
Exported as both `prisma` and `db` from package root.

## Key Models & Relationships

### Multi-Tenant Core

- **Site** → has many Domains, Pages, Brands (1:1), SEOConfig
  - `status`: DRAFT | REVIEW | ACTIVE | PAUSED | ARCHIVED
  - `targetMarkets`: string[] (country codes)
  - `primaryCurrency`: string (GBP, USD, EUR, etc.)
- **Domain** → belongs to Site
  - `status`: PENDING | REGISTERING | DNS_PENDING | SSL_PENDING | AVAILABLE | NOT_AVAILABLE | ACTIVE | ARCHIVED
- **Brand** → belongs to Site (1:1)
  - Colors, fonts, logos, tagline, isAutoGenerated

### Content System

- **Page** → belongs to Site or Microsite
  - Composite keys: `siteId_slug`, `micrositeId_slug`
  - `type`: HOMEPAGE | ABOUT | CONTACT | LEGAL | FAQ | BLOG | LANDING | CATEGORY | PRODUCT
  - `status`: DRAFT | PUBLISHED
  - `slug`: WITH prefix for BLOG/LANDING/FAQ, WITHOUT for CATEGORY/PRODUCT
- **Content** → belongs to Page (body, format)

### Microsites

- **MicrositeConfig** → extends Site for supplier/product/opportunity pages
  - `entityType`: SUPPLIER | OPPORTUNITY | PRODUCT
  - `layoutType`: Configurable layout variant
  - `status`: DRAFT | ACTIVE | ARCHIVED | PAUSED

### Booking & Analytics

- **Booking** → tracks booking lifecycle
  - `status`: PENDING | CONFIRMED | CANCELLED | FAILED
- **BookingFunnelEvent** → step-by-step journey tracking
- **PerformanceMetric** → time-series site performance data
- **ABTest** / **ABTestVariant** → experimentation infrastructure

### Background Jobs

- **Job** → tracks async operations
  - `type`: 80+ job types mapped to 11 queues
  - `status`: PENDING | RUNNING | COMPLETED | FAILED

### Platform Control

- **PlatformSettings** → global feature flags, rate limits, pause controls
  - `enableSiteCreation`, `enableContentGeneration`, `enableGSCVerification`, etc.

## Schema Conventions

- All IDs are `cuid()` strings
- Timestamps: `createdAt` (default now), `updatedAt` (auto-update)
- Soft deletes via status fields (ARCHIVED), not actual deletion
- Composite unique keys for multi-tenant isolation: `@@unique([siteId, slug])`
- JSON fields for flexible config (homepageConfig, seoConfig, brand colors)

## Common Pitfalls

1. **Slug queries** must account for prefix convention (BLOG stores `blog/my-post`, CATEGORY stores `food-tours`)
2. **Site filter**: Most queries need `where: { siteId }` for tenant isolation
3. **Microsite queries**: Use `micrositeId_slug` composite key, not `siteId_slug`
4. **Pool limit**: 4 connections — avoid long transactions or connection leaks
5. After schema changes, run `db:generate` AND check all downstream consumers
